## 🚀 使用有多简单？三步走

### 第一步：服务端搭好（IT 管理员做一次就行）

服务端部署在一个内网服务器上（比如 `192.168.22.212`），接收所有客户端上报的数据，统一存储和展示。

### 第二步：把客户端发给同事

一个单文件 exe，同事双击打开，看到的是这样一个极简界面：

```
┌──────────────────────────────┐
│   MQQ 终端资产采集客户端      │
│                              │
│   姓名：[        ]           │
│   部门：[        ]           │

│   ─────────────────────      │
│   服务器地址：192.168.22.212 │
│   API密钥：[******] [查看key]│
│                              │
│   请先测试服务器连通性 [测试] │
│   提示：关闭杀毒→管理员身份运行 │
└──────────────────────────────┘
```

### 第三步：填好姓名部门，一键完成

1. 输入姓名、部门
2. 选择设备归属（公司 / 私人）
3. 点"测试连通性"
4. **连通后自动采集上传，1 秒搞定，程序自动关闭**

不需要懂技术。不需要看命令行。填两个字段，点一下，完事。

## 💡 这个工具的设计巧思

### 1️⃣ 提权自动化

很多硬件信息（如主板型号、序列号）需要管理员权限才能读。程序启动时自动检测，没有管理员权限就**自动弹 UAC 窗口提权**，不需要手动右键"以管理员身份运行"。

### 2️⃣ 双重采集方案

优先走 WMI（Windows Management Instrumentation）直接读硬件信息，速度快、信息全。如果 WMI 取不到（比如系统精简过），**自动切到备用方案**继续采集，保证兼容性。

### 3️⃣ 智能识别组装机

品牌机通常有明确的制造商信息（如 Lenovo、Dell、HP），组装机则显示 "To Be Filled By O.E.M."。工具自动识别并在备注栏标注 **"疑似组装机"**，资产盘点一目了然。

### 4️⃣ 虚拟网卡自动过滤

一台电脑可能装了 VMware、Hyper-V，会有很多虚拟网卡。采集时**自动过滤**掉虚拟网卡、VPN 适配器、蓝牙适配器，只保留真实的有线网卡和无线网卡，MAC 地址准确无误。

### 5️⃣ 填完自动提交

不需要点"提交"按钮。只要服务器连通 + 姓名部门填好，**程序自动开始采集上传**，按钮变成滚动动画 "采集上传中 / - \ |"，上传成功 1 秒后自动关闭窗口。极致的"零操作感"。
